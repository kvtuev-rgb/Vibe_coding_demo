# Приложение заявок на участие в DnD

Небольшое веб-приложение для сбора заявок от игроков на участие в кампании Dungeons & Dragons (или других настольных ролевых играх).

## Возможности

- **Форма заявки**: имя/ник, контакты, опыт, идея персонажа, удобное время, предпочитаемые системы, комментарий мастеру.
- **Список заявок**: на той же странице мастер может развернуть блок со всеми отправленными заявками.
- **Хранение**: заявки сохраняются в базе данных **SQLite** (`data/applications.db`). Таблица создаётся автоматически при первом запуске.

## Как запустить

1. Установите зависимости:

   ```bash
   npm install
   ```

2. Запустите сервер:

   ```bash
   npm start
   ```

3. Откройте приложение в браузере:

   ```text
   http://localhost:3000
   ```

## Структура проекта

- `server.js` — Node/Express сервер, API для приёма и выдачи заявок, работа с SQLite.
- `public/index.html` — основная страница с формой и списком заявок.
- `public/main.js` — логика отправки формы и загрузки списка заявок.
- `public/styles.css` — оформление интерфейса (тёмная тема, аккуратные карточки).
- `data/applications.db` — база данных SQLite с таблицей заявок (создаётся автоматически при первом запуске).

## Заметки

- Это учебное/демо-приложение, без авторизации и сложных прав доступа.
- При желании можно:
  - добавить фильтры/поиск по заявкам;
  - вынести админскую часть на отдельный маршрут;
  - добавить миграции и более сложную схему БД.

## Деплой на Railway (через GitHub)

1. Создайте репозиторий на GitHub: `https://github.com/kvtuev-rgb/Vibe_coding_demo`.
2. В корне проекта выполните команды (один раз):

   ```bash
   git init
   git add .
   git commit -m "Initial DnD app"
   git remote add origin https://github.com/kvtuev-rgb/Vibe_coding_demo.git
   git push -u origin main
   ```

3. На сайте Railway создайте новый проект: **New Project → Deploy from GitHub repo** и выберите этот репозиторий.
4. Railway сам определит Node.js‑проект и выполнит `npm install`, затем `npm start`. Сервер уже слушает `process.env.PORT`, поэтому доп. настройки не нужны.
5. После деплоя Railway покажет внешний URL, по которому будет доступно приложение.

> Важно: SQLite‑база лежит внутри файловой системы контейнера (`data/applications.db`). При пересборке/удалении контейнера данные могут потеряться. Для боевого использования лучше вынести данные в постоянный Volume или внешнюю БД (например, Postgres на Railway).
